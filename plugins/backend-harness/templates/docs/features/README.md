# docs/features/

도메인별 상세 지식을 담는 **살아있는 문서** 모음이다. 코드만 봐서는 알 수 없는 것들 —
상태 머신, 외부 연동 제약, 과거 장애에서 배운 함정 — 을 기록한다.

- 파일명: `docs/features/<도메인>.md` (예: `paypal-payment.md`, `shopify-sync.md`)
- 소유: 해당 도메인을 마지막으로 만진 사람. 도메인 동작이 바뀌면 **같은 PR에서 갱신**한다.
- 작업 시작 전 참조: 해당 도메인 파일이 있으면 먼저 읽는다.
- 장애 대응 후: 배운 함정을 별도 분석 문서로 남기지 않고 바로 해당 파일의 "함정" 섹션에 추가한다.
- PLAN/장애 분석에서 오래 살아남을 지식은 머지 시 여기로 승격하고 원문은 삭제한다
  (`CLAUDE.md`의 "문서 체계 — 승격 규칙" 참조).

## 권장 구조 (도메인 파일 하나당)

```
# <도메인>

## 개요
## 상태 머신 / 라이프사이클
## 외부 연동 제약
## 함정 (Gotchas)
- QA-1234에서 승격: ...
```
