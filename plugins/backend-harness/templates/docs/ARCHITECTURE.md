## 모듈/서비스 지도 — 모듈러 모놀리스

하나의 배포 단위(모놀리스) 안에서 아래 모듈로 분리한다. 모듈 간 접근 규칙은
`.claude/rules/layer-architecture.md`의 "모듈 경계" 섹션이 강제한다.

| 모듈 | 책임 | 성격 | 비고 |
|---|---|---|---|
| `user` | 회원, 인증/인가, 프로필 | 도메인 | 다른 도메인 모듈이 참조하는 기반 모듈 |
| `album` | 앨범(키트) 콘텐츠 | 도메인 | |
| `community` | 커뮤니티(게시글·댓글 등) | 도메인 | `user`에 의존 |
| `order` | 주문 — Shopify 연동 | 도메인 + 외부 연동 | Shopify API는 별도 Client 클래스로 분리 (`EXTERNAL_API: true` 체크 대상) |
| `message` | 알림 발송 (이메일, 푸시 등) | 지원(supporting) | 도메인 모듈에 의존하지 않는다 — 도메인 이벤트를 구독해 발송 |
| `file` | 파일 업로드/저장 | 지원(supporting) | **Admin 서버 전용 배치 가능성 있음(미정)** — 결정 시 이 표와 토폴로지 갱신 |

- 의존 방향(초안): `community → user`, `order → user`. 그 외 도메인 모듈 간 직접 의존은
  기본 금지 — 필요해지면 이 표에 추가하고 이유를 남긴다.
- 지원 모듈(`message`, `file`)은 도메인 모듈을 역참조하지 않는다. 도메인 → 지원 방향만 허용
  (동기 호출 또는 이벤트).
- 아키텍처 스타일 결정(모듈러 모놀리스 + 선택적 헥사고날/DDD, Spring Modulith 경계 강제,
  모듈별 DB 소유권)은 **ADR-1** 참조. 모듈별 내부 구조 적용 수준(레이어드 vs 헥사고날)은
  `.claude/rules/layer-architecture.md`의 "모듈별 아키텍처 적용 수준" 표가 원본이다.
